Proyecto de Accesibilidad Web

Comparación entre una página sin accesibilidad y otra con accesibilidad completa

Este proyecto educativo demuestra la diferencia entre el desarrollo web sin considerar la accesibilidad y el desarrollo web siguiendo las mejores prácticas de accesibilidad. Ambas páginas tienen la misma funcionalidad y apariencia visual, pero difieren significativamente en su implementación técnica.


Funcionalidad común: Formulario de registro para cine con selección de películas por género, validación de campos, navegación responsive y visualización de resultados.

Página Sin Accesibilidad

Ejemplo de mala implementación con problemas de accesibilidad intencionados para fines educativos.

  • Solo usa DIV y SPAN
  • Sin elementos semánticos
  • Sin atributos ARIA
  • Outline de focus eliminado
  • Navegación personalizada bloqueada
  • Calendario personalizado complejo
  • Botones como DIVs

Página Con Accesibilidad

Implementación siguiendo las mejores prácticas de accesibilidad web y estándares WCAG.

  • Elementos semánticos HTML5
  • Atributos ARIA completos
  • Navegación por teclado estándar
  • Focus visible y accesible
  • Validación HTML5 nativa
  • Input type="date" nativo
  • Skip links y landmarks
